- D-05: isScrolling 标志 — 防止无限循环

- D-07: RAF 批处理 — requestAnimationFrame 保证 60fps

**研究中的关键实现模式:**
```
Row-based Index Sync:
1. Assign data-index to markdown source lines
2. Apply data-index to rendered elements (tr, li, not containers)
3. On scroll: find visible elements, scroll target to matching data-index

Infinite Loop Prevention:
- isScrolling flag prevents bidirectional feedback
- Only sync from "source" to "target", never both directions at same time
```

```typescript
import { useRef, useCallback, useEffect, RefObject } from 'react';

interface ScrollSyncOptions {
  /** Enable/disable sync (controlled externally) */
  enabled?: boolean;
  /** Sync direction: 'bidirectional' | 'source-to-target' | 'target-to-source' */
  syncDirection?: 'bidirectional' | 'source-to-target' | 'target-to-source';
  /** RAF batching for smooth 60fps */
  useRAF?: boolean;
}

interface ScrollSyncReturn {
  /** Ref to attach to source scroll container */
  sourceRef: RefObject<HTMLElement | null>;
  /** Ref to attach to target scroll container */
  targetRef: RefObject<HTMLElement | null>;
  /** Toggle sync on/off */
  setEnabled: (enabled: boolean) => void;
  /** Current enabled state */
  isEnabled: boolean;
}

/**
 * Bidirectional scroll synchronization hook
 * - Uses refs (not state) for DOM manipulation to avoid re-renders
 * - Uses RAF batching for smooth 60fps performance
 * - isScrolling flag prevents infinite feedback loops
 * - Row-based index sync via data-index attributes
 */
export function useScrollSync(options: ScrollSyncOptions = {}): ScrollSyncReturn {
  const {
    enabled = true,
    syncDirection = 'bidirectional',
    useRAF = true,
  } = options;

  // Refs for DOM elements
  const sourceRef = useRef<HTMLElement>(null);
  const targetRef = useRef<HTMLElement>(null);
  
  // isScrolling flag prevents infinite loop
  const isScrollingRef = useRef(false);
  
  // RAF handle for cleanup
  const rafHandleRef = useRef<number | null>(null);
  
  // Local enabled state
  const enabledRef = useRef(enabled);
  enabledRef.current = enabled;

  // Find element with matching data-index in target container
  const findMatchingElement = useCallback((
    targetContainer: HTMLElement,
    sourceIndex: string
  ): HTMLElement | null => {
    return targetContainer.querySelector(`[data-index="${sourceIndex}"]`);
  }, []);

  // Scroll target container to match source position
  const syncScroll = useCallback((
    sourceContainer: HTMLElement,
    targetContainer: HTMLElement
  ) => {
    if (!enabledRef.current) return;
    
    // Find visible elements with data-index
    const sourceElements = sourceContainer.querySelectorAll('[data-index]');
    const containerRect = sourceContainer.getBoundingClientRect();
    
    // Find first visible element (intersecting viewport)
    let firstVisibleIndex: string | null = null;
    for (const el of sourceElements) {
      const rect = el.getBoundingClientRect();
      if (rect.top >= containerRect.top && rect.top <= containerRect.bottom) {
        firstVisibleIndex = el.getAttribute('data-index');
        break;
      }
    }
    
    if (firstVisibleIndex) {
      const targetEl = findMatchingElement(targetContainer, firstVisibleIndex);
      if (targetEl) {
        targetEl.scrollIntoView({ behavior: 'instant', block: 'start' });
      }
    }
  }, [findMatchingElement]);

  // Handle scroll event with RAF batching
  const handleScroll = useCallback((
    sourceContainer: HTMLElement,
    targetContainer: HTMLElement
  ) => {
    if (!enabledRef.current || isScrollingRef.current) return;
    
    // Cancel any pending RAF
    if (rafHandleRef.current !== null) {
      cancelAnimationFrame(rafHandleRef.current);
    }
    
    // RAF batching
    if (useRAF) {
      rafHandleRef.current = requestAnimationFrame(() => {
        isScrollingRef.current = true;
        syncScroll(sourceContainer, targetContainer);
        isScrollingRef.current = false;
      });
    } else {
      isScrollingRef.current = true;
      syncScroll(sourceContainer, targetContainer);
      isScrollingRef.current = false;
    }
  }, [syncScroll, useRAF]);

  // Setup scroll listeners
  useEffect(() => {
    const sourceEl = sourceRef.current;
    const targetEl = targetRef.current;
    
    if (!sourceEl || !targetEl) return;

    // Determine which direction(s) to sync
    const shouldSyncSourceToTarget = syncDirection === 'bidirectional' || syncDirection === 'source-to-target';
    const shouldSyncTargetToSource = syncDirection === 'bidirectional' || syncDirection === 'target-to-source';

    const handleSourceScroll = () => {
      if (shouldSyncSourceToTarget) {
        handleScroll(sourceEl, targetEl);
      }
    };

    const handleTargetScroll = () => {
      if (shouldSyncTargetToSource) {
        handleScroll(targetEl, sourceEl);
      }
    };

    sourceEl.addEventListener('scroll', handleSourceScroll, { passive: true });
    targetEl.addEventListener('scroll', handleTargetScroll, { passive: true });

    return () => {
      sourceEl.removeEventListener('scroll', handleSourceScroll);
      targetEl.removeEventListener('scroll', handleTargetScroll);
      if (rafHandleRef.current !== null) {
        cancelAnimationFrame(rafHandleRef.current);
      }
    };
  }, [syncDirection, handleScroll]);

  // Expose control methods
  const setEnabled = useCallback((newEnabled: boolean) => {
    enabledRef.current = newEnabled;
  }, []);

  return {
    sourceRef,
    targetRef,
    setEnabled,
    isEnabled: enabledRef.current,
  };
}
```

Modify MarkdownPreview to add data-index attributes to rendered markdown elements for line mapping.

In the ReactMarkdown components prop, add data-index to these elements:
- `tr` (table rows) - use the row index
- `li` (list items) - use the list item index  
- `h1`, `h2`, `h3`, `h4`, `h5`, `h6` (headings) - use the heading text or index
- `p` (paragraphs) - use paragraph index
- `pre` (code blocks) - use the code block index

The data-index should be a string that matches between source and translated content. Use a sequential counter for each element type.

Implementation:
1. Add a counter ref that resets for each major element type
2. Assign data-index={`${type}-${counter}`} where type is 'tr', 'li', 'h1', etc.
3. For table rows (`tr`), increment on each row
4. For list items (`li`), increment on each item
5. For headings, use the heading level + index

Example output:
```jsx
<tr data-index="tr-0" ...>
<td data-index="tr-0" ...>content</td>
</tr>
<li data-index="li-0" ...>item</li>
<h1 data-index="h1-0" ...>Heading</h1>
```

Important: Apply data-index to the actual content elements (tr, li, p, h*), NOT to container elements. This follows D-08: "Apply data-index to rendered elements (tr, li, not containers)".
